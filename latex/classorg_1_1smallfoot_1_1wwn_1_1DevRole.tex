\subsection{Detailed Description}
This class seeks to provide simple bitfield type-\/checking of a device in a fabric so that I can use type-\/specific actions in a dependent project. 

By using this class to lift the assumption that all known devices are Targets, I can create additional possible matches that can be additionally checked when asked for more than default target types. This lets me later package the logic of subparts of a device (W\+W\+N\+N and W\+W\+P\+N) into the device logic itself, packaging up in a single place and avoiding needless diuplication without significantly stressing this model
